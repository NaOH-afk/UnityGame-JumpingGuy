# JumpingGuy

2D 横版平台跳跃游戏，使用 **Unity 2020.3.49f1c1** 开发。玩家控制角色在多个关卡中跑跳、收集樱桃、躲避陷阱，并支持通过串口与外部开发板联动控制。

## 游戏简介

JumpingGuy 是一款经典风格的 2D 平台跳跃游戏，共包含 **6 个关卡**，从主菜单出发，逐关挑战直至通关。场景中设有移动平台、粘性平台、旋转障碍等元素，踩中陷阱会扣除生命；收集樱桃可累计分数。

## 主要玩法

- **平台跳跃：** 左右移动、跳跃，穿越各类平台与障碍
- **樱桃收集：** 触碰樱桃获得分数，UI 实时显示收集数量
- **生命系统：** 初始 3 条生命，踩中陷阱扣 1 命；仍有生命时角色闪烁并弹起，命用尽后回出生点并重置为 3 条命
- **粘性平台：** 站在特定平台上会随平台一起移动
- **关卡推进：** 抵达关卡终点自动进入下一关

## 操作方式

### 键盘（默认）

| 操作 | 按键 |
|------|------|
| 向左移动 | `A` / `←` |
| 向右移动 | `D` / `→` |
| 跳跃 | `空格` |

### 串口控制（可选）

项目内置 `SerialController`，可通过串口接收开发板指令控制角色，默认配置为 **COM7、9600 波特率**（可在 Inspector 中修改）：

| 指令 | 功能 |
|------|------|
| `A` | 向左移动 |
| `B` | 向右移动 |
| `0` | 停止移动 |
| `C` | 跳跃 |

游戏还会向开发板发送状态指令：

| 指令 | 含义 |
|------|------|
| `[trap]` | 玩家失去 1 条生命 |
| `[R]` | 生命重置为 3 条（场景切换或命用尽复活时） |

> 无串口设备时游戏仍可正常运行，会自动尝试连接并在失败后周期性重试，不影响键盘操作。

## 场景结构

| 场景 | 说明 |
|------|------|
| `Assets/Scenes/Start.unity` | 主菜单 |
| `Assets/Scenes/Level1.unity` ~ `Level6.unity` | 六个游戏关卡 |
| `Assets/Scenes/End.unity` | 通关结束页 |

## 环境要求

- **Unity 版本：** 2020.3.49f1c1（或兼容的 2020.3 LTS 版本）
- **平台：** Windows（串口功能需在 Windows 下使用）

## 如何运行

1. 克隆本仓库：
   ```bash
   git clone https://github.com/NaOH-afk/UnityGame-JumpingGuy.git
   ```
2. 使用 Unity Hub 打开项目根目录。
3. 首次打开时 Unity 会自动重新生成 `Library` 等缓存目录，请耐心等待。
4. 打开 `Assets/Scenes/Start.unity`，点击 Play 从主菜单开始游戏。

## 主要脚本

| 脚本 | 功能 |
|------|------|
| `PlayerMovement.cs` | 角色移动、跳跃与动画状态 |
| `PlayerLife.cs` | 生命管理、陷阱碰撞与复活逻辑 |
| `SerialController.cs` | 串口通信与开发板指令同步 |
| `item_collector.cs` | 樱桃收集与计分 |
| `Finish.cs` | 关卡终点检测与场景切换 |
| `StickyPlatform.cs` | 粘性移动平台 |
| `StartMenu.cs` / `EndMenu.cs` | 菜单界面控制 |
| `AllContorl.cs` | 全局 GameManager（分数、生命） |
